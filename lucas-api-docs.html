LUCAS API

The LUCAS API provides programmatic access to physical asset verification. Request inspections, retrieve cryptographically signed proofs, and integrate real-world condition data into your applications and smart contracts.

Base URL

https://api.lucas.app/v1

Verified Data

GPS-stamped, cryptographically signed

On-chain Ready

Anchor to any EVM chain

Real-time

Webhooks for live updates

SDKs

JavaScript, Python, Go

Authentication

The LUCAS API uses API keys to authenticate requests. Include your key in the Authorization header of every request.

API Key Format

lcs_live_xxxxxxxxxxxxxxxx

Test keys use test prefix instead of live

Protect your API keys

Never expose keys in client-side code or commit them to version control.

Authenticated Request 
curl https://api.lucas.app/v1/proofs/PROOF-2024-0847 \
  -H "Authorization: Bearer lcs_live_xxxxxxxxxxxx"
Response Headers
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 99
X-Request-Id: req_abc123xyz

LUCAS Proofs

A LUCAS Proof is a cryptographically signed, immutable record of a physical asset's condition at a specific point in time. Each proof contains inspection data, photos, GPS verification, condition scores, and detected issues.

GET /proofs List all proofs
GET /proofs/{proof_id} Retrieve a proof
POST /proofs/request Request new inspection

The Proof Object

proof_id string Unique identifier
asset_id string Asset this proof belongs to
condition.score integer Score from 0-100
condition.grade string A, B, C, D, or F
issues array Detected issues list
signature.hash string SHA-256 proof hash
signature.anchor_tx string Blockchain tx hash

Webhooks

Receive real-time notifications when events occur. LUCAS sends POST requests to your configured endpoints with event data.

POST /webhooks Create webhook
GET /webhooks List webhooks
DELETE /webhooks/{id} Delete webhook

Event Types

proof.completed New proof ready
proof.anchored Proof written to chain
asset.score_changed Condition score updated
issue.detected New issue found
issue.resolved Issue marked fixed
Webhook Payload 
{
  "id": "evt_1234567890",
  "type": "proof.completed",
  "created": "2024-12-05T14:32:00Z",
  "data": {
    "proof_id": "PROOF-2024-0847",
    "asset_id": "asset_marina_1205",
    "condition_score": 94,
    "issues_found": 1
  }
}
Signature Verification 
// Verify webhook signature
const signature = req.headers['x-lucas-signature'];
const isValid = lucas.webhooks.verify(
  req.body,
  signature,
  webhookSecret
);

// Request a new inspection

  type: 'recurring_inspection',
  priority: 'standard',
  webhook_url: 'https://yourapp.com/webhooks'
});

API requests are rate limited based on your plan. Rate limit headers are included in every response.

Growth 500 req/min 1,000 req
Enterprise Custom Custom

Errors

LUCAS uses conventional HTTP response codes. Codes in the 2xx range indicate success, 4xx indicate client errors, and 5xx indicate server errors.

Code Description
200 Success
400 Bad request — invalid parameters
401 Unauthorized — invalid API key
404 Not found — resource doesn't exist
429 Rate limit exceeded
500 Server error — try again later
Error Response 401
